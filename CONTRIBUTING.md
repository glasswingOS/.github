## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md) (if you haven't added one yet, we recommend using the [Contributor Covenant](https://www.contributor-covenant.org/)).  
We expect all contributors to be respectful, inclusive, and constructive.

## How Can I Contribute?

### Reporting Bugs

If you find a bug:
1. Check the [existing issues](https://github.com/YOUR_ORG/glasswingos/issues) to see if it's already reported.
2. Open a new issue and include:
   - A clear, descriptive title
   - Steps to reproduce the bug
   - Expected behavior vs. actual behavior
   - Environment details (OS, language version, dependencies, etc.)
   - Any relevant logs or screenshots

### Suggesting Enhancements

We welcome ideas for new features, detection patterns, mitigation helpers, or integrations that strengthen defensive security capabilities.

1. Open an issue with the label **enhancement**.
2. Clearly describe the enhancement and why it would be useful to the community.
3. Provide as much detail as possible (use cases, proposed implementation, etc.).

### Submitting Code Changes

1. Fork the repository.
2. Create a new branch for your feature or fix (`git checkout -b feature/amazing-feature` or `git checkout -b fix/bug-name`).
3. Make your changes.
4. Add or update tests where appropriate.
5. Ensure the code follows the project's style and passes existing tests.
6. Commit your changes with clear, descriptive messages.
7. Push to your branch and open a Pull Request.
